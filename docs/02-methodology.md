# Part 2 - Development Methodology

## Which methodology will we follow?
The project will be developed following the agile methodology. Since the scope of the application is quite big, it will be developed part by part, and each part will be improved as time goes by. In order to follow the agile methodology within the scope of the project, the team will follow the Scrum framework for the development of the web application.

## Why is the chosen methodology appropriate for this project?
- **Requirements may change.** We are still figuring out many aspects of the system, such as wait time estimates. If we were to use the waterfall approach, we would have a large up-front investment before we even figured that out. Agile allows us to continue to adjust as we learn more about the project and its requirements.
- **We can build in small steps.** We will be using Scrum methodology with short sprints of a working piece of the project.
- **We can use feedback effectively.** After each submission, we can apply feedback from the professor or TAs to make adjustments to the project.

## How will this approach help us work across multiple assignments?
This is the most beneficial methodology to use for working across multiple assignments. Since the project is split into 4 assignments, we will be applying scrum to the assignments individually, treating each one as a sprint with it's own goal and deliverable.
